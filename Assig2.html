Overview

For this assignment, you will write three programs in three separate files, which you will hand in (the .java files, not the .class files).

In addition to those three files, you also need to down load the “IO Module”, i.e. the file IO,java, from the folder Sakai > Resources > IO module. This file gives you a number of predefined methods which you will use to read input from the keyboard and to print output in a form that our grading program can understand. Do not modify or hand in the IO module. Before you begin, read this document: How to use the IO module in programming assignments in the file io-tutorial.html which is also in the Resources folder Resources > IO module. Note that you do not have to understand how the methods in IO.java work, you just have to know what they do and how to call them.

Be sure to choose appropriate test cases to test each of your programs thoroughly, including error conditions. Test your program on all of your test cases - remember that a program that does the correct thing for one set of input data may not behave correctly for other sets of input data. You do not have to submit your test cases, however.

Problems

Problem 1

Write your program in the file Fine.java.

This program computes computes the overdue penalties (if any) on a library book when it is returned. The fee schedule is:

  • For every day late up to and including 7 days, the late fee is 10 cents per day.

  • Beyond 7 days, for every day late up to and including 90 days, the late fee is 20 cents per day (example: a book returned 10 days late incurs a late fee of $1.30 late fee – 10 cents for each of the first 7 days plus 20 cents for each of the remaining 3 days).

  • Beyond 90 days, an unreturned book is considered lost. All late fees are waived (i.e. are not charged), but the borrower is charged the cost of the book, plus a $10 restocking fee.

Ask the user for the number of days the book is late, and cost of the book in that order. Always ask for the cost of the book no matter what number of days late is given. Output one number, the fine as a number of cents. (So if the fine is $1.35 the program should print 135.) Do NOT use functions in the IO module print anything except this number.

Example:

java Fine
Days late?
7
Book cost?
2000
RESULT: 70

Note that "Days late?" and “Book cost?” must be printed using System.out.println, and the line "RESULT: ..." must be printed by IO.outputIntAnswer.

Problem 2

Write your program in the file TwoSmall.java.

Input a set of positive integers, ending with -1 as a sentinel. Print the smallest and second smallest of these numbers in that order.

You may assume there are always at least two numbers before the -1; you do NOT have to check for this being true.

Examples:

java TwoSmall
32
11
19
7
-1
RESULT: 7
RESULT: 11
java TwoSmall
20
34
20
-1
RESULT: 20
RESULT: 20

java TwoSmall
20
3
20
-1
RESULT: 3
RESULT: 20

In these examples, the lines in blue are typed by you and the lines that start "RESULT:" are printed by your program, using IO.outputIntAnswer.

Hint: Suppose the smallest number you have seen so far is 10 and the second smallest is 15. Now you see 5. What are the new smallest and second smallest numbers? Suppose instead of 5 you saw 12, what are the new smallest and second smallest? What if you saw 20 instead of 5?

Problem 3

Write your program in the file ReverseWords.java.

This program should read a line of text and print it back with each word reversed.

Example:

java ReverseWords

Four score and seven years ago

RESULT:  ruoF erocs dna neves sraey oga

Note that the line "RESULT: ruoF ..." must be printed using IO.outputStringAnswer.  You may assume that there is exactly one space between words in the input line, and no spaces before the first word or after the last. Any character other than space or upper or lower case letters is an error, which should cause your program to call IO.reportBadInput. You do not have to check or change the case of any letters.
